# 极简风格设计说明

## 设计理念

- **极简风格（Minimal）**: 黑白灰为主，大量留白
- **现代字体**: Inter, Space Grotesk - 细、冷感
- **记录类网站**: 非艺术网站，专注于内容展示
- **干净简洁**: 不喧宾夺主，突出内容本身

## 颜色系统

- **主色**: 纯白背景 `#ffffff`
- **文字**: 纯黑 `#000000`
- **次要文字**: 灰色系 `#525252`, `#737373`, `#a3a3a3`
- **边框**: 浅灰 `#e5e5e5`, `#d4d4d4`
- **无彩色强调**: 仅使用黑白灰，无其他颜色

## 字体系统

- **正文**: Inter (300-700 weight)
- **标题**: Space Grotesk (300-700 weight)
- **字重**: 主要使用 300 (light), 400 (regular), 500 (medium)
- **字间距**: -0.01em (紧凑)

## 导航结构

顶部固定导航，包含 4 项：

1. **On-site / 在现场** - 实地探索记录
2. **Stories / 日志** - 随笔、装备、感想
3. **Gallery / 图库** - 图片展示
4. **About / 关于** - 个人介绍

导航特点：
- 英文为主，中文小字标注
- 悬停效果极简
- 当前页面高亮

## 页面结构

### 首页 (Homepage)

1. **Hero Section**
   - 大标题：`Urban Exploration Notes`
   - 副标题：`Capturing forgotten spaces and the stories they left behind.`
   - 极简，大量留白

2. **On-site 快速入口**
   - 显示最近 6 个地点
   - 卡片：标题、封面图、时间、描述
   - Grid 布局（响应式）

3. **Highlight Stories**
   - 展示 2-3 个精选故事
   - 文本为主，极简卡片
   - 左标题右时间

4. **Gallery 预览**
   - 一行 6 张横向小图
   - 点击进入 Gallery 全页

5. **Footer**
   - About 链接
   - 联系方式
   - Copyright
   - 站点说明

### On-site / 在现场 页面

- **顶部标题**: On-site / 在现场
- **Zones 分类按钮**:
  - All
  - Industrial
  - School
  - Residential
  - Hospital
  - Others
- **Grid 展示**: 所有地点
  - 地点名
  - 简介
  - 封面图
  - 时间

### Stories / 日志 页面

- **标题**: Stories / 日志
- **按时间排序**: 最新在前
- **卡片样式**:
  - 纯白背景
  - 左标题右时间
  - 可加小图（非必须）
  - 文本为主

### Gallery / 图库 页面

- **瀑布流布局**: 响应式多列
- **点击 Lightbox**: 全屏查看
- **键盘导航**: 左右箭头切换
- **简洁不喧宾夺主**

### About / 关于 页面

- 头像/现场照
- 简短自我介绍
- 为什么喜欢废墟探索
- 联系方式（邮箱）
- 网站说明

## 组件特点

### 卡片 (Card)
- 纯白背景
- 浅灰边框
- 悬停时边框变深
- 无阴影或极淡阴影

### 链接 (Link)
- 下划线仅在悬停时显示
- 颜色变化极简
- `underline-offset-4` 间距

### 按钮 (Button)
- 极简边框
- 文本为主
- 无背景色（或极淡）

## 留白策略

- **大间距**: 使用 `py-16`, `py-24` 等大间距
- **Section 间距**: 使用 `border-t` 分隔
- **内容区域**: `max-w-4xl` 或 `max-w-7xl` 限制宽度
- **内边距**: `px-6 lg:px-8` 响应式内边距

## 响应式设计

- **移动端**: 单列布局
- **平板**: 2 列布局
- **桌面**: 3 列布局
- **断点**: Tailwind 默认断点

## 图片处理

- 所有图片使用占位符
- 统一 `aspect-ratio` 保持比例
- 悬停效果极简
- 懒加载（Next.js Image）

## 代码特点

- **组件化**: 每个功能独立组件
- **类型安全**: TypeScript
- **可维护**: 清晰的目录结构
- **可扩展**: 易于添加新内容

## 数据管理

- 使用静态数据文件
- 易于替换为 CMS
- 类型定义完整
- Placeholder 内容便于替换

---

**设计原则**: 极简、现代、专注内容、大量留白、黑白灰为主
