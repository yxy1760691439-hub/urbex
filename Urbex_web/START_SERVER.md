# 🚀 启动开发服务器

## 方法 1: 在终端中手动启动（推荐）

1. **打开终端**（Terminal）

2. **进入项目目录**：
   ```bash
   cd /Users/yuxinyuan/Desktop/Project/Urbex_web
   ```

3. **启动开发服务器**：
   ```bash
   npm run dev
   ```

4. **等待编译完成**，你会看到类似这样的输出：
   ```
   ▲ Next.js 14.x.x
   - Local:        http://localhost:3000
   - Ready in X.Xs
   ```

5. **打开浏览器**，访问：
   ```
   http://localhost:3000
   ```

## 方法 2: 在 VS Code 中启动

1. 打开 VS Code 终端（`Ctrl + ~` 或 `Cmd + ~`）
2. 运行 `npm run dev`
3. 等待编译完成
4. 点击终端中的链接或手动访问 http://localhost:3000

## 常见问题

### 端口被占用

如果 3000 端口被占用，Next.js 会自动使用下一个可用端口（如 3001）。

查看终端输出，找到实际使用的端口。

### 编译错误

如果看到编译错误：
1. 查看终端中的错误信息
2. 检查是否有语法错误
3. 确保所有依赖已安装：`npm install`

### 服务器启动慢

首次启动可能需要较长时间（1-2分钟），这是正常的。

后续启动会更快。

## 停止服务器

在运行 `npm run dev` 的终端中按：
- `Ctrl + C` (Mac/Linux)
- `Cmd + C` (Mac)

## 查看日志

服务器运行时的所有日志都会显示在终端中，包括：
- 编译状态
- 错误信息
- 访问日志

---

**提示**: 如果自动启动失败，请手动在终端运行 `npm run dev`。
